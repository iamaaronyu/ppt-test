What Is tmux?

一个终端,
无限可能。

tmux 是一个终端复用器(Terminal Multiplexer),它让你在单个终端窗口中管理多个会话。断开 SSH 连接后,程序继续运行——重新连上,一切如初。

"tmux 让你可以在远程服务器上开一个持久化的工作区,关掉笔记本、换个地方、重新 SSH 上去——所有窗口、分屏、运行中的程序都还在。"
— tmux 官方手册

Act I · 03 / 09
Three Pillars · 三大支柱

Session · Window · Pane

tmux 的层级模型极其清晰——理解这三个概念,就理解了 tmux 的全部。

Layer 1 · 最外层
Session
会话是最顶层的容器。一个 Session 包含多个 Window。你可以 detach(分离)一个 Session,断开连接后它仍在后台运行——这就是 tmux 最核心的价值。
Layer 2 · 中间层
Window
窗口类似于浏览器 Tab。每个 Window 占据整个屏幕,你可以在不同 Window 之间切换。一个 Session 可以管理数十个 Window。
Layer 3 · 最内层
Pane
面板是 Window 内的分屏。你可以把 Window 切成左右、上下、任意组合的 Pane,每个 Pane 运行独立的 shell——实现真正的"一屏多用"。

Prefix Key · 前缀键

一切始于
Ctrl+B

tmux 所有快捷键都以前缀键 Ctrl+B 开始。按下前缀键后释放,再按功能键——两步操作,防止快捷键冲突。你也可以把它重映射为 Ctrl+A(GNU Screen 用户的最爱)。

"记住 Ctrl+B, ? ——这条命令列出所有快捷键。你不需要背下来,需要时查就好。"
— veteran tmux user

Before / After · 效率对比

有了 tmux 之后,一切都变了

Before · 没有 tmux

多窗口地狱

  • SSH 断开 → 所有任务丢失
  • 需要开 N 个终端窗口来回切换
  • 无法在一个屏幕里同时看日志 + 编辑代码
  • 长任务跑一半,网络抖动全白费
  • 团队协作靠"你帮我看看这台机器"
After · 有了 tmux

一个终端,一切尽在掌控

  • SSH 断开重连 → 会话完好如初
  • 一个终端窗口管理所有 Session
  • 分屏:左看日志,右写代码,顶跑监控
  • 长任务安心跑,关电脑也不中断
  • Pair programming:两人接入同一 Session

Use Cases · 五个经典场景

每个人都在用 tmux 做什么？

远程开发
SSH 到开发机,在 tmux Session 中打开 Vim、数据库客户端、日志监控。网络断了也不怕——重新连上,一切复原。
长期监控
htop 看资源、tail -f 追日志、watch 定时刷新——三个 Pane 并排,系统状态尽收眼底。关掉终端,监控继续。
结对编程
两人同时 attach 到同一个 tmux Session,输入同步显示——最轻量的远程协作方案,不需要任何额外工具。
部署与运维
在生产服务器上开一个 tmux Session 执行部署。即使网络抖动,部署流程不会中断——这是运维的基本素养。
工作区持久化
用 tmux 脚本或插件(如 tmux-resurrect)保存完整工作区布局。重启机器后一键恢复所有窗口和分屏。
多任务并行
Window 1 跑测试,Window 2 写代码,Window 3 看文档——一个终端,多个上下文,Ctrl+B, N 秒切。

进阶技巧 · Pro Tips
Act III · 08 / 09
Beyond the Basics

进阶技巧

01
插件生态

TPM(tmux Plugin Manager)一键安装插件。tmux-sensible 合理默认值、tmux-resurrect 会话持久化、tmux-continuum 自动保存——社区已经把轮子造好了。

02
鼠标支持

set -g mouse on 开启鼠标模式——点击切换 Panel、拖拽调整分屏大小、滚轮翻页。终端操作瞬间现代化。

03
美化与配置

定制状态栏(显示时间/CPU/电池)、更换配色方案、用 tmuxp 写 YAML 配置文件一键启动复杂工作区布局。
