What & Why

终端复用器

Tmux（Terminal Multiplexer）让终端会话与终端窗口解耦，实现会话持久化、多窗口管理、多窗格分屏、远程开发保活，甚至多人共享会话。

对于服务器开发、AI 训练、远程调试、Claude Code、SSH 运维等场景，Tmux 几乎是必备工具。

Act I · 03 / 12
Before / After

为什么需要 Tmux？

Before · 普通终端

关闭终端 = 丢失一切

  • ssh server → python train.py
  • 网络断开 → SSH 超时 → 训练终止
  • 本地电脑关机 → 所有进程退出
  • 需要开 N 个终端窗口来回切换
  • 无法在一个屏幕同时看日志 + 编辑
After · 有了 Tmux

终端窗口 ⟂ 运行会话

  • SSH 断开重连 → tmux attach 恢复现场
  • 训练任务在后台持续运行，不受网络影响
  • 关电脑也不怕——Session 在服务器上活着
  • 一个终端窗口管理所有 Session
  • 分屏：左看日志、右写代码、顶跑监控
Terminal → Tmux Session → Shell → Task | Terminal 关闭 ≠ Session 消失

Window · 窗口

一个 Session,
多个 Window。

Window 就像浏览器里的 Tab 页——每个 Window 占满整个屏幕，Ctrl+B, C 新建，Ctrl+B, N/P 切换，Ctrl+B, W 浏览全部。

典型场景：Window 1 跑测试，Window 2 写代码，Window 3 看日志——一个终端，多个上下文，秒切不卡。

Prefix Key · 前缀键

一切始于 Ctrl+B

Tmux 默认前缀键为 Ctrl+B。所有快捷键都分两步——先按前缀键并释放，再按功能键。防止快捷键冲突。

最常用的 5 个命令

tmux ls

tmux attach -t work

tmux kill-session -t work

Act III · 10 / 12
Troubleshooting

常见问题与解决

Problem 01

出现 "sessions should be nested with care" 错误

原因：当前已在一个 Tmux Session 内部，又尝试 tmux attach。

✅ 推荐方案
tmux switch-client -t 目标会话
或: Ctrl+B, S 选择目标 Session
Problem 02

找不到会话: "no server running"

没有存活的 Tmux Session。可能被重启清理了，或从未创建过。

✅ 解决方案
tmux ls
先查看，若没有就重新创建: tmux new -s work
Pro Tip

鼠标不能用？

默认 Tmux 不开启鼠标。需要在 ~/.tmux.conf 中加一行配置。

set -g mouse on
开启后可以点击切换 Pane、拖拽调整大小、滚轮翻页
Quick Check

如何判断当前是否在 Tmux 中？

一个命令即可判断：

echo $TMUX
有输出 → 在 Tmux 中 | 空 → 裸终端

Claude Code + Tmux

Claude Code
推荐用法

iTerm2 / Terminal→ SSH→ tmux→ Claude Code / Log / Monitor / Shell
启动工作区
tmux new -s claude claude
Window 1 · AI 编程
并行监控
Ctrl+B, C tail -f server.log
Window 2 · 日志
随时恢复
Ctrl+B, D tmux attach -t claude
SSH 断开也不怕

即使 SSH 断开，Ctrl+B, D 分离后，Claude Code 会话完好无损。
重新登录 tmux attach -t claude 即可恢复全部工作现场。
