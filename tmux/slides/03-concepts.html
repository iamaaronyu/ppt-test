Act I · 核心概念

Core Concepts

三大支柱

tmux 的层级模型极其清晰。理解这三个概念，就理解了 tmux 的全部——从最外层的持久会话，到最内层的分屏操作。

核心哲学

每一层都是上一层「状态的容器」。Session 承载 Window，Window 承载 Pane——分离时整棵树原地存活。

Layer 1 · 最外层

Session

会话是最顶层的容器。一个 Session 包含多个 Window。你可以 detach（分离）一个 Session——断开连接后它仍在后台运行。这就是 tmux 最核心的价值。

Layer 2 · 中间层

Window

窗口类似于浏览器 Tab。每个 Window 占据整个屏幕，你可以在不同 Window 之间切换。一个 Session 可以管理数十个 Window。

Layer 3 · 最内层

Pane

面板是 Window 内的分屏。可以把 Window 切成左右、上下、任意组合的 Pane，每个运行独立的 shell——实现真正的「一屏多用」。
